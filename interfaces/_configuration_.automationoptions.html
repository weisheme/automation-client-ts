Optional policy

policy: "ephemeral" | "durable"
  • Defined in configuration.ts:120

If events should be queued when the registration is not connected to the websocket, specificy "durable". "ephemeral" is suited for testing and running locally and is the default.

Optional postProcessors

postProcessors: Array<function>
  • Defined in configuration.ts:180

Post-processors can be used to modify the configuration after all standard configuration loading has been done and before the client is started. Post-processors return a configuration promise so they can be asynchronous.

Optional teamIds

teamIds: string[]
  • Defined in configuration.ts:106

Atomist teams this automation will be registered with. Must be specified if groups is not specified. Cannot be specified if groups is specified.

Optional token

token: string
  • Defined in configuration.ts:128

GitHub personal access token with, at minimum, read:org scope. The GitHub user that owns this token must be associated with an Atomist person with the developer role. Additional scopes may be necessary if the automation uses the token to perform actions against the GitHub API.

Optional version

version: string
  • Defined in configuration.ts:100

Automation version. Must be a valid semantic version, https://semver.org/. If not given, the version is extracted from the package.json.
